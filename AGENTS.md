# Agent Instructions

## Core Rules
- Follow project architecture and conventions
- Prefer simple, maintainable solutions
- Always use RED GREEN TDD
- Do not make destructive changes without approval

## Documentation
- /docs contains committed project documentation
- shared personal skills and references are provided through the configured skills system

## UI Work
- Use the ui-design skill
- Read `/docs/desgin-langauge.md` before implementing or revising app UI so new work matches the established component language
- Use `/docs/desgin-langauge.md` as the local source of truth for component hierarchy, tile states, spacing, and copy restraint
